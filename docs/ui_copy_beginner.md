# UI文言セット（初心者向け）
本ドキュメントは、入力コントロールの**ラベル名 / 補足説明 / プレースホルダ / ツールチップ / 警告文**をまとめたものです。  
（Next.js 16 + shadcn/ui想定）

---

## 1. 画面タイトル・セクション
- 画面タイトル：**建玉と損切りのおすすめ計算**
- サブタイトル：**勝率と平均損益から、無理のない1回の取引量を提案します**

### モード選択
- ラベル：**モード**
- 選択肢：
  - **優待イベント投資（株）**
  - **先物・FX（Backtesting.py）**
- 補足：**モードによって初期設定（安全寄り度合い）が変わります**

---

## 2. 基本入力（必須）

### E0（初期資金）
- ラベル：**はじめの資金（円）**
- プレースホルダ：例）1000000
- 補足（短）：**運用を始める元手です**
- ツールチップ（長）：**この金額を基準に、1回で許容する損失（円）と建玉額を計算します**

### L（停止ライン）
- ラベル：**停止ライン（%）**
- UI例：スライダー + 入力（%）
- デフォルト表示：50%
- 補足（短）：**資金がここを下回ったら停止します**
- ツールチップ（長）：**資金が「はじめの資金×この%」を下回ったら、運用停止（強制終了）とみなします。高すぎるとすぐ停止しやすく、低すぎると大きな損失を許容する設定になります。**

### p（勝率）
- ラベル：**勝率（%）**
- プレースホルダ：例）55
- 補足（短）：**1回の取引が勝つ割合です**
- ツールチップ（長）：**Backtesting.pyのWin Rate（%）に近い値を入れてください**

### W%（平均利益）
- ラベル：**勝ったときの平均利益（%）**
- プレースホルダ：例）1.8
- 補足（短）：**勝ったときに平均で何%増えるか**
- ツールチップ（長）：**「1トレードあたりの%」として入力してください（複利・レバ込みの結果でもOK）**

### D%（平均損失）
- ラベル：**負けたときの平均損失（%）**
- プレースホルダ：例）1.2
- 補足（短）：**負けたときに平均で何%減るか（正の数）**
- ツールチップ（長）：**マイナス記号は不要です。例：-1.2%なら「1.2」と入力します**

### 最大ドローダウン（DD）
- ラベル：**最大ドローダウン（DD）（%）**
- プレースホルダ：例）8.3
- 補足（短）：**最大の下落幅（%）**
- ツールチップ（長）：**DDが分かる場合は入力すると、1回の最大損失は計算値とDD換算の小さい方になります**

---

## 3. 詳細設定（折りたたみ：任意）

### N（想定トレード回数）
- ラベル：**想定トレード回数（回）**
- プレースホルダ：例）200
- 補足（短）：**この回数の中で起きそうな連敗を見積もります**
- ツールチップ（長）：**回数が多いほど「長い連敗が起きる前提」になり、安全寄りの推奨になりやすいです**

### α（停止ラインを割る確率の目安）
- ラベル：**停止ラインを割る確率の目安（%）**
- プレースホルダ：例）1.0
- 補足（短）：**小さいほど安全寄り**
- ツールチップ（長）：**これは“保証”ではなく目安です。入力値を使って、想定連敗を保守的に見積もるために使います**

### k（攻めすぎ防止）
- ラベル：**攻めすぎ防止（成長の理論値を抑える）**
- UI例：スライダー（0.05〜0.5など）
- 補足（短）：**小さいほど安全寄り**
- ツールチップ（長）：**理論上の最適（ケリー）をそのまま使うとリスクが大きくなりがちなので、控えめにするための係数です**

### s_loss（損切りゆとり）
- ラベル：**損切りゆとり（平均損失に上乗せ）**
- UI例：1.0〜2.0
- 補足（短）：**平均損失より少し広めに損切りを見積もります**
- ツールチップ（長）：**平均損失は“たまたま小さく見える”ことがあります。少しゆとりを持たせることで、過度な建玉になりにくくします**

### f_user_max（1回の最大損失）
- ラベル：**1回の最大損失（資金に対して%）**
- プレースホルダ：例）0.3
- 補足（短）：**どんな場合でも、1回でこれ以上は失わない上限**
- ツールチップ（長）：**計算結果が大きく出ても、この上限を超えないように抑えます**

## 4. 結果表示（初心者向け）

### メイン結果
- 見出し：**おすすめ**
- カード1：
  - ラベル：**建玉（投入額）の目安**
  - 表示：`¥1,234,567`
  - 補足：**この金額で建てると、損切りしたときの損失が設定内に収まる想定です**
- カード2：
  - ラベル：**損切り幅の目安**
  - 表示：`1.8%`
  - 補足：**平均損失に「損切りゆとり」をかけて決めています**
- カード3（セットで表示すると安心感が出ます）：
  - ラベル：**1回で最大いくら負ける？**（DD入力時は**最大DD換算の想定損失**）
  - 表示：`¥3,000`
  - 補足：**計算上の最大損失とDD換算の小さい方を表示します**

### 詳細（折りたたみ）
- 見出し：**計算の中身（詳細）**
- 表示項目：
  - **資金に対する1回の最大損失（%）**：`f%`
  - **ペイオフ比（利益÷損失）**：`b`
  - **どの上限が効いたか**：`ケリー抑え / 停止ライン / ユーザー上限`
  - **想定連敗回数（m）**：`m`

## 5. 警告・注意文（短く、怖がらせない）
- 注意（固定表示）：  
  **この推奨は利益や安全を保証するものではありません。入力値と相場状況により結果は変わります。**

### 入力が極端なとき（警告）
- 勝率が極端：  
  **勝率が極端です。入力値（Backtesting.pyの統計）を確認してください。**
- 平均損失が小さすぎる：  
  **平均損失が小さすぎる可能性があります。損切りゆとりを上げると安全寄りになります。**
- 停止ラインが高すぎる：  
  **停止ラインが高いと、少しの下落で停止になりやすいです。**

## 6. ボタン文言
- 「計算する」→ MVPでは自動計算なら不要。置くなら：**計算する**
- 「リセット」：**入力をリセット**
- 「保存」：LocalStorage自動なら不要。置くなら：**この設定を保存**
- 「詳細を開く」：**詳細を表示**
